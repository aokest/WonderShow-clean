# Dashboard 设计令牌（v0.7 录制工作室）

> 本文件是 Dashboard 视觉与交互语言的**权威源**，与 `docs/modules/dashboard/DESIGN.md`、`SPEC.md` 配合阅读。
> 任何代码与 Figma 设计都必须使用本文件中的令牌，禁止私自引入新值。
> 2026-06-19 注：当前已确认稳定的 `v0.7.20260619 (202606190305)` 使用暖黑金录制工作室界面。早期 v0.5 的冷黑/青色令牌只作为历史约束参考；新增按钮、弹窗、源选择器、导出进度、mini toolbar 和未来授权弹窗，应优先贴合现有暖黑金界面，而不是恢复旧冷色风格。

## 1. 设计意图

- 目标：让一名讲师/主播在 5 秒内看清「画面、下一步、状态」三件事。
- 风格坐标：**广播导播室 × 编辑极简**。靠近 OBS / DaVinci Resolve / Logic Pro 的「安静侧」。
- 明确反对：赛博朋克、玻璃拟态、渐变网格、紫蓝 AI 发光、大色块情绪插画。

## 2. 颜色

### 2.1 中性表面（Surface）

| Token              | Hex       | 用途                                  |
|--------------------|-----------|---------------------------------------|
| `--bg-base`        | `#0B0D10` | 窗口底色                              |
| `--bg-raised`      | `#14171C` | 卡片底色                              |
| `--bg-overlay`     | `#1B1F26` | 输入控件、悬浮菜单、按钮 hover        |
| `--border-subtle`  | `#232730` | 1px 分隔线、卡片描边                  |
| `--border-strong`  | `#2D323B` | 输入控件描边、激活态描边              |

### 2.2 文本（Text）

| Token              | Hex       | 用途                                  |
|--------------------|-----------|---------------------------------------|
| `--text-primary`   | `#E8E9ED` | 主标题、数值、按钮文字                |
| `--text-secondary` | `#9CA0A8` | 次级描述、Caps 标签                   |
| `--text-tertiary`  | `#5E636C` | 时间戳、版本号、占位                  |
| `--text-disabled`  | `#5E636C` | 禁用态（与 tertiary 同色 + 40% 透明） |

> 严禁使用纯白 `#FFFFFF` 或纯黑 `#000000`。

### 2.3 状态色（State，只用于状态，不做装饰）

| Token              | Hex       | 用途                                  |
|--------------------|-----------|---------------------------------------|
| `--state-live`     | `#3FB6BD` | 已连接、识别中、待机但就绪            |
| `--state-rec`      | `#D8463E` | 录制进行中（仅 REC 按钮与 REC 状态）  |
| `--state-warn`     | `#D4A14A` | 权限缺失、设备异常、低优先级告警      |
| `--state-success`  | `#5BA873` | 校准完成、保存成功等一次性正向反馈    |

> 每屏同色状态色最多出现 2 次，避免成为「装饰色」。

## 3. 字体

| 用途        | 字体                | 字重              | 备注                              |
|-------------|---------------------|-------------------|-----------------------------------|
| UI 主体     | `Inter`             | 400 / 500 / 600   | 系统回退：`-apple-system`         |
| 数据/时间码 | `JetBrains Mono`    | 500               | 用于时间码、设备 ID、版本号       |
| 中文        | `PingFang SC`       | 跟随 Inter 字重   | 自动通过系统字体回退              |

### 3.1 排印阶梯

| 名称         | size / line-height / weight        | 用途                          |
|--------------|------------------------------------|-------------------------------|
| `display`    | 28 / 32 / 600                      | 暂未启用，预留给未来 splash   |
| `title`      | 15 / 20 / 600                      | 卡片标题、按钮                |
| `body`       | 13 / 20 / 400                      | 普通正文、Detail Line value   |
| `caption`    | 12 / 16 / 400                      | 次要说明、按钮副标            |
| `label`      | 11 / 14 / 600，caps + tracking 0.08em | 状态标签、表单 label        |
| `mono-data`  | 13 / 20 / 500，tabular-nums         | 时间码、设备 ID、状态值       |

> 所有数据类文本必须开启 `font-feature-settings: "tnum"`，保证位数对齐。

## 4. 间距与圆角

- 4 / 8 / 12 / 16 / 24 / 32 / 48（4 的倍数）
- 卡片内边距：16
- 卡片间距：16
- 区域外边距：24
- 圆角：6（chip）/ 8（按钮、输入）/ 12（卡片）/ 16（预览大块）
- 严禁出现 < 6 的圆角，禁止 > 16 的圆角。

## 5. 阴影与描边

- 卡片：1px `--border-subtle` 实线描边 + 内嵌 1px `rgba(255,255,255,0.04)` 高光。
- 浮层菜单：0 8px 24px `rgba(0,0,0,0.4)` + 1px `--border-subtle`。
- 录制状态条：2px `--state-rec`，顶部进度条专用，**禁止**用于其他场景。
- 禁止使用彩色阴影、霓虹光晕、模糊半径 > 32 的雾化效果。

## 6. 动效

| 场景            | 时长     | 曲线                              |
|-----------------|----------|-----------------------------------|
| Hover           | 150ms    | `ease-out`                        |
| 面板展开        | 240ms    | `cubic-bezier(0.2, 0.8, 0.2, 1)`  |
| 录制点呼吸      | 1200ms   | `ease-in-out` infinite            |
| 手势热区激活    | 200ms    | `ease-out`                        |
| 全局转场        | 240ms    | `cubic-bezier(0.2, 0.8, 0.2, 1)`  |

- 必须尊重 `prefers-reduced-motion`：关闭呼吸与展开动画，仅保留 80ms opacity 渐隐。
- 禁止弹簧动画、视差、粒子。

│ │                          │  │ Quick start             │     │
│ │     16:9 Preview         │  ├─────────────────────────┤     │
│ │  [LIVE]   [chips chips]  │  │ Presentation            │     │
│ │                          │  ├─────────────────────────┤     │

- 三段固定高：56 / 自适应 / 40。
- 预览 16:9，左列占 60%，右列 40%。
- 右列 4 张卡片等高（flex），不单独成节。
- 状态条 4 个 pill 等宽。

## 8. 关键组件契约

> 所有组件必须与下方命名一致，便于 Figma 与 SwiftUI 双向同步。

| 组件           | 关键属性                                                                 |
|----------------|--------------------------------------------------------------------------|
| `StatusPill`   | icon(14, stroke 1.75) · label(11 caps) · value(13/500) · live dot(6)    |
| `PrimaryButton`| height 32 · padX 14 · radius 8 · bg `--state-rec` 或 `--bg-overlay`     |
| `SecondaryButton` | height 32 · padX 14 · radius 8 · bg `--bg-raised` · 1px `--border-strong` |
| `Card`         | radius 12 · 1px `--border-subtle` · bg `--bg-raised` · pad 16            |
| `DetailLine`   | label(11 caps secondary) · value(13/500 primary) · 单行 28 高            |
| `SegmentedChip`| height 32 · radius 8 · 激活：bg overlay + 1px state-live 描边            |
| `FormRow`      | 11px caps label 顶部 + 控件下方，gap 8                                   |
| `LiveDot`      | 6px 圆点 · 录制 1.2s 呼吸 · 待机常亮 · 离线 6px hollow                  |

## 9. 反模式清单（每次评审必查）

- 渐变背景
- 玻璃模糊卡片
- 紫蓝/青色 AI 发光
- 卡片头部使用大色块图标
- Emoji 出现在 UI 中
- 大色条 "Insight Tile"（旧版设计已废弃）
- 一屏出现 3 种以上状态色
- 圆角 < 6 或 > 16
- 弹簧 / 视差 / 粒子动效
- 任何 Tailwind 调色板风的「500/600/700 彩虹」
