# Dashboard 测试要点

## 手工检查

1. 启动应用后应直接看到主预览区
2. 未连接摄像头时应看到明确提示与重连按钮
3. 切换演示目标后，状态区应同步更新
4. 彩排和录制按钮应能切换状态文案
5. 录制按钮应支持开始、暂停、继续、终止保存/放弃，并清零计时
6. 录制源选择器应显示真实活跃窗口/屏幕，支持列表和缩略图
7. 音频输入选择应能显示系统默认和可用麦克风
8. 监视器中拖拽/缩放/改变画中画形状后，预览合成和导出应一致
9. 录制中切换布局后，预览合成和导出应出现对应时间段的布局变化
10. 高级诊断默认应折叠，展开后可见权限与投递信息

## 回归重点

- 小白用户能否快速找到主按钮
- 错误提示是否能直接给出下一步动作
- 预览叠加信息是否过多，是否遮挡画面
- 已稳定主链路不要被 UI 调整破坏：录制源、音频、画中画、布局 keyframe、预览、导出都要复测
- 未来 `⌘1` 到 `⌘6` 快速切源要测试热键冲突、源位编号反馈、窗口关闭后的降级和录制中 raw 轨同步
