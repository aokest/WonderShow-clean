# Dashboard 设计说明

## 设计目标

- 让首页首先服务“看画面、开始彩排、开始录制、确认手势状态”
- 降低技术信息噪音
- 让用户知道当前系统“能不能用、为什么不能用、下一步做什么”

## 布局原则

- 左侧以大预览为中心
- 右侧用卡片承载任务分组
- 高风险或低频信息放入高级诊断区

## 视觉方向 (v0.7.20260619 稳定基线)

- 采用深邃的“暖黑金”色盘（背景 `#0D0A07`，表面 `#181309`，高光 `#C8923A`，边框 `#3A2E1E`）。
- 去工程化：消除生硬的大色块卡片，通过精细的边框 `stroke`、内部渐变、内阴影和高斯模糊营造质感。
- 层次分明：预览画面占据视觉中心，右侧面板使用可折叠卡片以保持呼吸感。顶部使用胶囊式状态指示器。
- 文本层级严格遵守设计稿规范，次级文字和边框融入暗色背景，突出核心交互按钮。
- 当前稳定界面已经包含录制源选择、音频输入选择、画中画拖拽/缩放/形状、预览合成、导出进度和录制状态控制；后续 UI 调整必须延续这一套暖黑金语言，不要另起一套组件风格。
- 未来快捷切源需要在源选择器、顶部状态、监视器角标和 mini toolbar 中保持一致的源位反馈，避免用户只靠记忆 `Command+1` 到 `Command+6`。

## 关键状态

- 摄像头连接
- 手势引擎状态
- 当前目标应用
- 最近动作
- 权限与投递诊断
- 当前录制源和用户自定义源位
- 录制中暂停/继续/终止状态
- 画中画尺寸、位置、形状与导出一致性
