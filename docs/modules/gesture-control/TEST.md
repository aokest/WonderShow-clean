# 手势控制测试要点

## 单元测试

- 热区判断
- 开掌停留识别
- 解锁窗口超时
- 冷却期间拒绝重复动作
- 双手普通摆动不应被连续缩放候选提前吞掉
- 只有双手都满足缩放手型时，才应优先进入连续缩放分支
- 多手入镜时应选择热区内最靠近中心的两只手
- 单手握拳抓取移动时应持续输出平移，松开后停止

## 手工验证

1. 把手放到热区外，翻页不应触发
2. 画面中出现其他人的手时，只要自己的手在热区内，翻页与缩放仍应可用
3. 开掌停留后，系统应进入可触发状态
4. 在窗口内做左挥，应该翻到下一页
5. 连续快速挥动，不应连续多次翻页
6. 双手缩放停手后不应继续持续放大/缩小（阻尼收敛）
7. 双手缩放过程中如果手型变化（例如握拳），应立刻停止缩放跟踪
8. 已缩放状态下，单手握拳并移动应平移缩放中心；松开握拳应固定当前位置
9. 已解锁状态下，单手握拳应单步放大；单手张开应单步缩小
10. 双手连续缩放时，测试页动画应表现为连续跟随，不应明显“一格一格跳”
11. 当 MediaPipe 出现短时空帧时，后端仍应继续尝试 MediaPipe；不能因为短时空帧让双手缩放整体退化成 Vision 单手翻页
12. 当最近窗口连续出现 `lShape,lShape` 时，应优先进入连续缩放链路，不能继续触发翻页
13. 连续双手缩放时，连续两次 `setZoom` 的差值不应出现夸张跳变；测试页视觉上应表现为细腻连续，而不是大幅脉冲
14. 双手八字保持期间，不应同时触发上一页/下一页
15. 双手八字刚稳定形成后的前 1 到 2 帧内，也不应漏出一次上一页/下一页误触
16. 做很慢的小幅双手缩放时，应看到细小连续缩放；做明显更快的双手分开/合拢时，应看到更快但仍受控的等比例缩放
17. HTML 测试页缩放动画不应再出现明显弹簧式过冲、回摆或左右晃动
18. 右挥与左挥都应在较短动作窗口内被识别，不应出现“下一页稳定、上一页总要更大幅度才触发”的明显偏置
19. `Pointing_Up` 和 `Victory` 手型的代表锚点应由多个 landmark 加权而成，不能完全依赖单个指尖点，否则应在单元测试中暴露为锚点抖动风险
