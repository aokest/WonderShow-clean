# 手势控制规格

## 当前支持动作

- 开掌停留：进入可触发状态
- 左挥：下一页
- 右挥：上一页
- 双手分开：放大
- 双手合拢：缩小
- 单手握拳：在已解锁状态下，触发单步放大（默认 15%）
- 单手张开：在已解锁状态下，触发单步缩小（默认 15%）
- 单手握拳并移动：在已缩放状态下，拖拽移动缩放中心；松开握拳即固定当前位置

## 门控规则

- 只有中央热区内的手势才参与判定
- 画面中出现多只手时，系统只会选择热区内最靠近画面中心的最多两只手作为“当前操作者手”，其余手会被忽略（用于应对画面内出现其他人）
- 未解锁时，翻页和缩放不应直接触发
- 解锁窗口超时后回到待命状态
- 指令触发后进入冷却期
- 双手缩放只有在两只手都满足缩放手型时才优先占用输入，不能用普通双手摆动吞掉翻页识别
- 当最近识别窗口已经稳定出现双手八字时，应直接优先进入连续缩放链路，不能继续被过严的运动阈值阻塞
- 双手缩放只在两只手都保持八字手型时持续生效，手型变化（例如握拳）应立刻停止缩放跟踪
- 缩放到位后采用阻尼式收敛（避免停手后持续放大/缩小）
- 连续缩放应限制单次目标缩放变化幅度，避免在短时间内从 100% 跳到 130%+ 的夸张动画
- 双手缩放姿态稳定出现时，应抑制离散翻页识别，避免双手八字误触上一页/下一页
- 缩放姿态抑制离散翻页时，必须把“当前刚进入的这一帧”也纳入判断，不能只看旧窗口；否则双手八字刚稳定时仍会漏出一次误翻页
- 连续缩放应按双手距离变化幅度分级响应：细小位移使用更小步长，明显快速位移使用更大步长，但单次更新仍需受上限约束
- 连续缩放应优先追求“连续跟手”而非保守滞后：在 M2 本机环境下，MediaPipe 采样间隔可适当前移，以提升接近触屏的顺滑度
- 指向/八字等关键手型的运动锚点不应只依赖单个固定点位；应按手型使用多个 landmark 计算稳定锚点，减少某一根手指抖动带来的翻页和缩放迟滞
- 双手八字缩放不要求先解锁（因为手型本身足够明确）；单手缩放与拖拽仍要求先开掌停留解锁
- MediaPipe 单帧或短时空帧只应视为“当前帧未识别”，不能因此整段降级到 Vision；否则会丢失双手缩放能力

## 数据输出

- 当前手型
- 当前会话状态
- 最近动作
- 当前缩放值
- 热区状态

## 替换边界

- UI 层不直接依赖具体识别框架
- 未来替换为 MediaPipe 时，保留相同的手势事件输出语义
