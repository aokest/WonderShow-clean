# 风险模型

## 产品风险

- 误触翻页会直接打断演讲节奏
- 权限未授权会让用户误以为产品失效
- 设备连接不稳定会影响预览与录制准备

## 技术风险

- Vision 单帧结果抖动，导致姿态分类不稳定
- 主线程负载过高，导致预览卡顿和手势延迟
- 不同摄像头视角差异大，单一阈值不通用

## 交互风险

- 将调试信息直接暴露给普通用户，形成认知负担
- 手势动作过多，用户难以记忆且难以形成稳定习惯
- 缺少“当前是否可触发”的可视反馈，导致盲操作

## 风险缓解

- 采用两阶段交互：先解锁，后执行
- 加入中央热区与冷却期
- 在 UI 中明确展示识别状态、可触发状态和最近动作
- 为高风险操作保留按钮兜底，不强依赖手势

## 回滚策略

- Dashboard 保持单文件回滚能力
- 手势稳态控制与底层检测解耦，可单独关闭
- 识别异常时自动退回“仅预览 + 手动控制”
