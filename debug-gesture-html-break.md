# [OPEN] gesture-html-break

## 问题摘要

- 现象 1：手势识别依然不准
- 现象 2：个人校准完成后，测试 HTML 页面完全没反应
- 现象 3：UI 视觉糟糕，文字与背景对比度不足，调试信息暴露过多

## 本轮目标

- 收集运行时证据，确认 HTML 页面无响应的真实原因
- 收集运行时证据，确认手势识别链路中误判和漏判的主要环节
- 在有证据后给出最小修复，再进入更成熟的 AI 手势算法替换方案

## 初始假设

1. 手势会话门控新增后，解锁状态没有正确进入，导致根本没有向命令层发出翻页动作。
2. 校准流程改变了手势模板库状态，导致离散手势识别被模板库“锁死”或始终判空。
3. HTML 测试页命令发送了，但 `DemoControlServer` 或本地桥接没有收到、没有广播、或页面监听断开。
4. HTML 测试页命令根本没发送，因为 `PresentationCommandController.handle()` 没被触发，或目标路由被错误切换。
5. 手势不准的核心原因不是阈值，而是 Vision 只取单锚点导致特征信息严重丢失，需要切换为更成熟的 21 点时序模型。

## 当前状态

- 状态：等待插桩
- 约束：在拿到运行时证据前，不修改业务逻辑
