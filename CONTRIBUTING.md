# Contributing

Thank you for improving WonderShow Core.

Good first contribution areas:

- schema validation improvements;
- importer/exporter examples;
- MediaPipe sidecar compatibility tests;
- plugin API feedback;
- documentation and sample project files.

Before opening a pull request:

1. Run `swift test`.
2. Keep changes inside the public package boundary.
3. Do not include private app code, credentials, signing assets, or paid product assets.
4. Describe whether your change affects the public schema.

Large API changes should start as an issue or proposal so the commercial app can stay compatible.
